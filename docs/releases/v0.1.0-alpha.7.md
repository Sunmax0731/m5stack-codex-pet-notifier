# v0.1.0-alpha.7

## 位置づけ

`v0.1.0-alpha.7` は closed alpha prerelease です。`v0.1.0-alpha.6` の pet animation FPS 改善に加え、M5Stack 上の pet 描画を Sprite buffer 経由にして、animation tick 中のちらつきを抑えます。stable release ではありません。

## 追加実装

- firmware の pet surface を `M5Canvas` off-screen Sprite へ描画し、完成した surface を `pushSprite()` で一括転送するようにした。
- pet animation tick では `needsPetRedraw` だけを立て、Answer / Choice / Notification の本文や footer を毎フレーム再描画しないようにした。
- scale-specific hatch-pet asset の run-length drawing も描画先を `Target` に統一し、Sprite 合成中に一部だけ直接 display へ描かれないようにした。
- `scripts/validate.mjs` に Sprite buffer、pet-only redraw、direct display bypass 禁止の source gate を追加した。
- manual test、GUI 手順、QCDS、hardware evidence、traceability に no-flicker 確認項目を追加した。

## 手動テスト準備

- Host Bridge 起動: `cmd.exe /d /s /c npm run bridge:start -- --host=0.0.0.0 --port=8080`
- Dashboard: `http://127.0.0.1:8080/`
- hatch-pet asset: `cmd.exe /d /s /c npm run pet:asset -- --pet-dir %USERPROFILE%\.codex\pets\Mira --output firmware\include\pet_asset.local.h`
- Core2 upload: `E:\DevEnv\PlatformIO\venv\Scripts\pio.exe run -d firmware -e m5stack-core2 -t upload --upload-port COM4`
- Display settings: `cmd.exe /d /s /c npm run codex:display -- --pet-scale 8 --ui-text-scale 2 --body-text-scale 2 --animation-fps 20`
- Pet update: `cmd.exe /d /s /c npm run codex:pet -- --name "Codex Pet" --state celebrate`

期待結果:

- Core2 の pet surface が固定ヘッダーテキストなしで表示される。
- pet frame / bounce が継続して動く。
- `20fps` にしても画面全体の黒塗り、Answer / Choice の本文、footer text の明滅が起きない。
- pet display area `1/8`、`4/8`、`8/8` で scale-specific frame が選択され、低解像度 base frame の単純拡大に見えない。

## 未実施範囲

GRAY 実機、長時間運用、実 Codex App 内部 API 連携の手動テストはCodexでは未実施です。Core2 への build / upload は実行対象ですが、Sprite buffer によるちらつき抑制の物理画面目視、Dashboard 最新回答送信、Codex session auto relay、Codex hook relay、Dashboard からの Choice / Pet 実機目視はユーザー手動確認対象です。
