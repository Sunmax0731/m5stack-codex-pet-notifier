# v0.1.0-alpha.10

## 位置づけ

`v0.1.0-alpha.10` は closed alpha prerelease です。`v0.1.0-alpha.9` のDashboard/Bridge改善に対して、8080番に古いHost Bridgeが残っていても最新Bridge APIを検出し、local hatch-pet asset previewを表示できるようにした修正版です。stable release ではありません。

## 修正内容

- Dashboard が同一originのAPIだけに固定されないようにし、`/debug/runtime` と `/health` のversionを見て最新Bridge APIを自動検出するようにした。
- 8080番に古いBridgeが残っている場合でも、`http://127.0.0.1:18081` などの最新Bridgeへ切り替え、`/pet/packages`、`/pet/current/manifest`、`/pet/current/spritesheet.webp` を取得できるようにした。
- spritesheet URLが相対パスで返る場合も、Dashboard側で検出済みBridge originへ解決するようにした。
- 最新Bridge側にCORS/OPTIONS応答を追加し、8080で開いたDashboardから18081のAPIとassetを安全に取得できるようにした。
- runtime sidebarと `debug JSON` linkも検出済みBridge APIに追従するようにした。
- Dashboard の topbar から旧ヘッダー文言とBridge行を削除し、操作ボタンの高さを統一した。
- 画面背景、pet 背景、文字色、文字背景、文字枠の操作を、色と透明度を同時に扱う項目別 RGBA picker へ統一した。
- `Pet 更新を送信` でも display 設定を同梱し、pet X/Y位置、画面背景、pet背景、文字背景が実機側へ届くようにした。
- command modal の重複していた `保守` tab を削除し、sample replay は `デバッグ送信` tab へ統合した。
- 文字背景 alpha が `0` の場合は firmware が text panel を塗らず、M5GFX の glyph background fill も無効化して文字だけを描くようにした。
- `M5Stack 表示プレビュー` 上で pet をドラッグして X/Y 位置を調整できるようにし、位置リセットボタンを追加した。
- pet display area を `1..32` に拡張し、`32/32` では現在の最大表示からさらに4倍の超拡大構図を確認できるようにした。

## 手動テスト準備

- 8080番の古いBridgeを閉じられる場合: `cmd.exe /d /s /c npm run bridge:start:bg -- --host=0.0.0.0 --port=8080`
- 8080番を閉じられない場合: `cmd.exe /d /s /c npm run bridge:start:bg -- --host=127.0.0.1 --port=18081`
- Dashboard: `http://127.0.0.1:8080/`
- `M5Stack 表示プレビュー` の `current pet` が `Mira` になり、fallback previewではなくMiraのspritesheet画像が表示されることを確認する。
- sidebarのruntime表示に `API: http://127.0.0.1:18081` 相当の警告が出る場合は、8080の画面から18081の最新Bridgeを使っている状態です。

## 未実施範囲

手動テストはCodexでは未実施です。Codex側ではCore2の再flashは未実施です。2026-05-09の最終リトライ時点ではM5Stack USB serialがWindowsに見えていなかったため、実機画面でのMira asset再確認はユーザー手動確認対象です。

GRAY 実機、長時間運用、実 Codex App 内部 API 連携の手動テストもCodexでは未実施です。
