# v0.1.0-alpha.4

## 位置づけ

`v0.1.0-alpha.4` は closed alpha prerelease です。`v0.1.0-alpha.3` の Codex relay answer 表示に加え、Core2 firmware の日本語表示を改善します。stable release ではありません。

## 追加実装

- Core2 / GRAY firmware で M5GFX の `fonts::efontJA_12` を使い、日本語を含む UTF-8 文字列を表示できるようにした。
- Answer 本文のページングと折り返しを UTF-8 code point 境界に変更し、日本語の途中バイトで `substring` しないようにした。
- Answer / Notification / Choice の本文領域をピクセル幅ベースで折り返すようにした。
- simulator の回答ページングも `Array.from()` を使った Unicode code point 単位へ変更した。
- `scripts/validate.mjs` に firmware 日本語フォント / UTF-8 境界 source gate を追加した。
- Host Bridge 再起動で pairing token が失効した場合、firmware が token を破棄して自動再pairingへ戻るようにした。
- repo root からの PlatformIO 実行手順を `pio.exe run -d firmware ...` に修正した。

## 手動テスト準備

- Host Bridge 起動: `cmd.exe /d /s /c npm run bridge:start -- --host=0.0.0.0 --port=8080`
- 日本語 answer: `cmd.exe /d /s /c npm run codex:answer -- --summary "日本語表示" --text "これは日本語の表示確認です。Core2のAnswer画面で文字化けせずに表示されれば合格です。"`
- event log: `Invoke-RestMethod -Uri http://127.0.0.1:8080/events`
- 期待結果: Core2 の Answer 画面に日本語の要約と本文が文字化けせず表示される。

## 未実施範囲

GRAY 実機、長時間運用、実 Codex App 内部 API 連携、日本語 glyph のユーザー目視確認の手動テストはCodexでは未実施です。Codex App の非公開内部 API scraping は意図的に対象外とし、clipboard / stdin / file relay を実アダプタとして使います。
