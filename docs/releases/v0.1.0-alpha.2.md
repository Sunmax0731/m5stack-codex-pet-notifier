# v0.1.0-alpha.2

## 位置づけ

`v0.1.0-alpha.2` は closed alpha prerelease です。`v0.1.0-alpha.1` で残っていた「LAN に接続している M5Stack へ実装済み firmware を入れて Host Bridge と通信する」範囲を MVP に含めます。stable release ではありません。

## 追加実装

- PC 側 LAN Host Bridge server を `src/host-bridge/server.mjs` として追加。
- `/pair`、`/device/poll`、`/device/event`、`/codex/event`、`/codex/replay-samples`、`/health`、`/events` を実装。
- Host Bridge smoke test を `scripts/bridge-smoke.mjs` と `npm run bridge:smoke` に追加。
- M5Unified firmware を Wi-Fi、pairing、HTTP polling、screen state、reply、pet interaction、heartbeat まで実装。
- Core2 / GRAY の PlatformIO build target を維持し、Core2 upload と LAN 接続確認を release evidence に含める。
- `docs/host-bridge-manual-check.md` を追加し、手動確認手順を独立化。

## 検証結果

- `cmd.exe /d /s /c npm test`
- `cmd.exe /d /s /c npm run bridge:smoke`
- `E:\DevEnv\PlatformIO\venv\Scripts\pio.exe run -e m5stack-core2`
- `E:\DevEnv\PlatformIO\venv\Scripts\pio.exe run -e m5stack-gray`
- `E:\DevEnv\PlatformIO\venv\Scripts\pio.exe run -e m5stack-core2 -t upload --upload-port COM4`

## 手動テスト準備

- Host Bridge 起動: `cmd.exe /d /s /c npm run bridge:start -- --host=0.0.0.0 --port=8080`
- sample replay: `Invoke-RestMethod -Method Post -Uri http://127.0.0.1:8080/codex/replay-samples -ContentType application/json -Body '{"deviceId":"m5stack-sample-001"}'`
- event log: `Invoke-RestMethod -Uri http://127.0.0.1:8080/events`

## 未実施範囲

GRAY 実機、物理 A/B/C、Core2 touch、GRAY IMU、実 Codex adapter の手動テストはCodexでは未実施です。手順は `docs/manual-test.md` と `docs/host-bridge-manual-check.md` に残しています。
