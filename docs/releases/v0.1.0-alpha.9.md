# v0.1.0-alpha.9

## 位置づけ

`v0.1.0-alpha.9` は closed alpha prerelease です。`v0.1.0-alpha.8` のMVP baselineに対して、実機表示のRGBA反映、Dashboardのコンパクト化、GUIからの環境構築/デバッグコマンド実行、Host Bridgeのバックグラウンド運用を追加した検証版です。stable release ではありません。

## 追加実装

- firmware のRGBA入力を object / array / `#rrggbb` / `#rrggbbaa` / CSV 形式で受けられるようにし、pet background、text color、text background のalpha合成を実機描画に反映するようにした。
- local hatch-pet asset の透明ピクセルが固定アクセント色で塗られないようにし、設定された pet background が見えるようにした。
- `M5Stack 表示プレビュー` を2カラム構成へ圧縮し、プレビュー、現在のpet情報、表示設定、RGBA設定を同一ペイン内で操作できるようにした。
- `環境構築コマンド` modal を環境構築 / デバッグ送信 / 保守のtab構成にし、Host Bridge起動、pet asset生成、Core2 upload、Codex answer / decision / display、latest session送信、sample replayをGUIから任意パラメータで実行できるようにした。
- `/debug/runtime`、`/debug/commands`、`/debug/commands/run` を追加し、Dashboard sidebarにHost Bridgeのforeground/background、pid、uptimeを表示するようにした。
- `bridge:start:bg` を追加し、WindowsでPowerShell画面を残さずHost Bridgeをバックグラウンド起動できるようにした。
- Dashboard smoke / validation に command modal、runtime sidebar、allowlist command実行、RGBA firmware gate、background launcher gateを追加した。

## 手動テスト準備

- Host Bridge 起動: `cmd.exe /d /s /c npm run bridge:start:bg -- --host=0.0.0.0 --port=8080`
- Dashboard: `http://127.0.0.1:8080/`
- hatch-pet asset: `cmd.exe /d /s /c npm run pet:asset -- --pet-dir %USERPROFILE%\.codex\pets\Mira --output firmware\include\pet_asset.local.h`
- Core2 upload: `cmd.exe /d /s /c npm run firmware:upload:core2`
- Display settings: GUIの `M5Stack 表示プレビュー` で pet background / text color / text background とalphaを変更し、`表示設定を送信` を押す。
- Command modal: sidebarの `環境構築コマンド` を開き、環境構築 / デバッグ送信 / 保守 tabから対象コマンドを実行する。
- Decision request: `cmd.exe /d /s /c npm run codex:decision:wait -- --question "次の作業を選んでください" --a "進める" --b "修正する" --c "保留する" --wait-ms 300000`

期待結果:

- Core2 のpet背景色、本文文字色、本文背景色、alphaがDashboard設定と一致する。
- local hatch-pet assetの透過部分は固定色ではなく、設定したpet backgroundとして見える。
- Dashboard previewは従来より低い高さで表示され、設定フォームと同時に確認できる。
- Command modalの各tabからallowlist commandが実行され、実行結果JSONがmodal内に表示される。
- Host Bridgeを `bridge:start:bg` で起動した場合、PowerShell画面を残さず、Dashboard sidebarに `Bridge running / background` とpidが表示される。

## 未実施範囲

手動テストはCodexでは未実施です。2026-05-09の最終リトライ時点ではM5Stack USB serialがWindowsに見えておらず、`npm run firmware:upload:core2` は `No M5Stack USB serial port was detected` で停止しました。M5StackをUSB接続し直してからCore2 uploadと物理画面のRGBA反映確認を実施してください。

GRAY 実機、長時間運用、実 Codex App 内部 API 連携の手動テストもCodexでは未実施です。GUIとfirmwareの自動検証、Core2/GRAY build、Dashboard browser smokeは通過済みです。
