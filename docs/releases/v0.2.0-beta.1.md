# v0.2.0-beta.1

## 位置づけ

`v0.2.0-beta.1` は beta prerelease です。`v0.1.0-alpha.10` の表示同期と Dashboard 改善を土台に、alpha 境界を越えて pet の表情、Core2 touch gesture、M5Stack から Codex へ三択を求める長押し workflow、Windows 導入用 installer package を追加します。stable release ではありません。

## 主な変更

- `pet.updated.pet.mood` を追加し、`idle / listening / thinking / happy / surprised / confused / sleepy / worried / alert / proud` の表情を Host、Simulator、Dashboard、Firmware で共通化した。
- Core2 の tap、double tap、long press、swipe up/down/left/right を `device.pet_interacted` として送信し、長押し時は Host Bridge が `prompt.choice_requested` を自動 queue する。
- M5Stack heartbeat に pet mood、lastInteraction、interactionCount を追加し、Dashboard の preview readout で実機側の最新操作を確認できるようにした。
- hatch-pet asset は標準 9 行 atlas を firmware header に取り込み、local asset では図形 marker を重ねず、キャラクターイラストの表情 / 姿勢 row で mood / state / touch gesture を表現する。vector fallback では従来の簡易 expression / marker を維持した。
- `start-dashboard.bat` を hidden launcher 経由に変更し、ダブルクリックで background Bridge 起動とブラウザ表示まで行うようにした。8080 番に古い Bridge が残っている場合は version を照合し、18081 以降の fallback port で beta Bridge を開く。
- Foreground の `bridge:start` も 8080 番が使用中の場合に Node stack trace を出さず、既存 Bridge version と fallback 手順を表示するようにした。
- Dashboard の表示設定 render で `rgbaLabel` helper の import 漏れにより F12 console error が出る問題を修正した。
- Core2 firmware の回復可能な poll 失敗は次の正常 poll で `Idle` へ戻し、`device.heartbeat` に `lastError` と `errorRecoverable` を返して Dashboard の実機反映欄で切り分けできるようにした。
- `installer/M5StackCodexPetNotifier-Setup.bat` と `installer/install-windows.ps1` を追加し、Desktop / Start Menu shortcut と install manifest を user local に作る installer を用意した。
- `tools/package-beta.mjs` と `npm run installer:package` を追加し、release asset 用の `dist/m5stack-codex-pet-notifier-v0.2.0-beta.1-windows-installer.zip` を生成できるようにした。
- Dashboard preview に pet mood、最新 interaction 表示、hatch-pet row preview を追加し、Pet 更新 payload に display 設定と mood を同梱する。
- 他リポジトリや他ユーザーへ配布できる M5Stack Choice Gate 用 `AGENTS.md` / `SKILL.md` / portable JSON block と `npm run choice:package` を追加した。
- GRAY 実機と GRAY IMU を release target 外に整理し、Core2 firmware target と Dashboard の button reference preview を分離した。
- 長時間運用に向けて Host Bridge の queue/log 上限、stale diagnostics、heartbeat age、dropped event count、firmware の HTTP timeout / Wi-Fi reconnect backoff / poll backoff / pairing 復帰を追加した。
- Codex App Server public interface 連携準備として `src/codex-adapter/appServerAdapter.mjs`、`npm run codex:app-server:smoke`、`npm run adapter:review` を追加し、非公開 API scraping を禁止する adapter review を追加した。
- 署名付き MSI / MSIX 準備として WiX template、MSIX manifest template、`npm run installer:signing:check` を追加した。

## 手動テスト

Core2 実機と Dashboard の手動確認は、機能ブラッシュアップ時にユーザーが動作を確認し、問題なく動作することを報告済みです。既存の upload / Wi-Fi / pairing / Codex answer 表示に加えて、以下を確認済みとして扱います。

- Dashboard で mood を `happy`、`confused`、`alert`、`sleepy` に切り替えて `ペット更新を送信` し、local hatch-pet asset では jumping / review / failed などのイラスト row が変わる。fallback pet の場合は簡易 marker / 表情差分で近似される。
- Core2 の pet を single tap / double tap / long press / swipe し、Dashboard event log と preview readout に `device.pet_interacted` の interaction、target、mood が出る。
- long press 後に `prompt.choice_requested` が queue され、Core2 の A/B/C で Codex 側へ `device.reply_selected` を返せる。
- `start-dashboard.bat` と installer shortcut から PowerShell 画面を残さず Dashboard が開く。

GRAY 実機と GRAY IMU は release target 外です。長時間 soak、実署名 MSI / MSIX は未確認です。実 Codex App Server 接続は beta 後の formal release 準備で runtime probe により `initialize`、`thread/start`、`turn/start` まで確認済みです。Codex App の非公開内部 API scraping は引き続き対象外とし、clipboard / stdin / file / local session JSONL relay、Codex hook relay、Codex App Server public interface adapter を実 adapter 境界として扱います。
