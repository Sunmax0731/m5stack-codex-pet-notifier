# v0.2.0-beta.1

## 位置づけ

`v0.2.0-beta.1` は beta prerelease です。`v0.1.0-alpha.10` の表示同期と Dashboard 改善を土台に、alpha 境界を越えて pet の表情、Core2 touch gesture、M5Stack から Codex へ三択を求める長押し workflow、Windows 導入用 installer package を追加します。stable release ではありません。

## 主な変更

- `pet.updated.pet.mood` を追加し、`idle / listening / thinking / happy / surprised / confused / sleepy / worried / alert / proud` の表情を Host、Simulator、Dashboard、Firmware で共通化した。
- Core2 の tap、double tap、long press、swipe up/down/left/right を `device.pet_interacted` として送信し、長押し時は Host Bridge が `prompt.choice_requested` を自動 queue する。
- M5Stack heartbeat に pet mood、lastInteraction、interactionCount を追加し、Dashboard の preview readout で実機側の最新操作を確認できるようにした。
- hatch-pet asset と vector fallback の両方で mood marker / expression を描画し、既存の Sprite buffer によるちらつき抑制を維持した。
- `start-dashboard.bat` を hidden launcher 経由に変更し、ダブルクリックで background Bridge 起動とブラウザ表示まで行うようにした。8080 番に古い Bridge が残っている場合は version を照合し、18081 以降の fallback port で beta Bridge を開く。
- Foreground の `bridge:start` も 8080 番が使用中の場合に Node stack trace を出さず、既存 Bridge version と fallback 手順を表示するようにした。
- Dashboard の表示設定 render で `rgbaLabel` helper の import 漏れにより F12 console error が出る問題を修正した。
- `installer/M5StackCodexPetNotifier-Setup.bat` と `installer/install-windows.ps1` を追加し、Desktop / Start Menu shortcut と install manifest を user local に作る installer を用意した。
- `tools/package-beta.mjs` と `npm run installer:package` を追加し、release asset 用の `dist/m5stack-codex-pet-notifier-v0.2.0-beta.1-windows-installer.zip` を生成できるようにした。
- Dashboard preview に pet mood、最新 interaction 表示、mood marker を追加し、Pet 更新 payload に display 設定と mood を同梱する。

- `cmd.exe /d /s /c npm run dashboard:smoke`: pass

- `cmd.exe /d /s /c npm test`: release 前に再実行し、`dist/validation-result.json`、`dist/dashboard-smoke-result.json`、`dist/m5stack-codex-pet-notifier-docs.zip`、installer zip を更新する。

## 手動テスト

手動テストはCodexでは未実施です。Core2 実機では、既存の upload / Wi-Fi / pairing / Codex answer 表示に加えて、以下を重点確認してください。

- Dashboard で mood を `happy`、`confused`、`alert`、`sleepy` に切り替えて `ペット更新を送信` し、実機 pet の表情または mood marker が変わる。
- Core2 の pet を single tap / double tap / long press / swipe し、Dashboard event log と preview readout に `device.pet_interacted` の interaction、target、mood が出る。
- long press 後に `prompt.choice_requested` が queue され、Core2 の A/B/C で Codex 側へ `device.reply_selected` を返せる。
- `start-dashboard.bat` と installer shortcut から PowerShell 画面を残さず Dashboard が開く。

GRAY 実機、GRAY IMU、長時間運用、実 Codex App 内部 API 連携は未確認です。Codex App の非公開内部 API scraping は引き続き対象外とし、clipboard / stdin / file / local session JSONL relay と Codex hook relay を実 adapter として扱います。

## 未解決リスク

- Core2 の gesture 目視、mood marker 目視、long press から Choice 自動 queue の実機確認はユーザー手動。
- Windows installer は user-local shortcut installer であり、署名付き MSI / MSIX ではない。
- stable release までに GRAY 実機、長時間運用、複数 M5Stack 同時接続、installer 署名、実 Codex App 公開 API が利用可能になった場合の adapter 見直しが必要。
