# v0.1.0-alpha.5

## 位置づけ

`v0.1.0-alpha.5` は closed alpha prerelease です。`v0.1.0-alpha.4` の日本語表示対応に加え、Windows PowerShell から clipboard 本文を読む経路の文字化けを修正します。stable release ではありません。

## 追加実装

- `tools/generate-pet-firmware-asset.py` を追加し、`%USERPROFILE%\.codex\pets` の hatch-pet package を firmware 用 RGB565 local header に変換できるようにした。
- firmware は `firmware/include/pet_asset.local.h` がある場合に hatch-pet asset を優先表示し、未生成時は vector fallback avatar を表示する。
- `firmware/include/pet_asset.local.h` を `.gitignore` 対象にし、個人 pet sprite を release asset に含めないようにした。
- `codex:sessions` を追加し、`%USERPROFILE%\.codex\sessions` の最新 Codex session JSONL から最近の user / assistant のやり取りを opt-in で M5Stack へ自動送信できるようにした。
- `codex:hook` と `docs/codex-hooks.example.json` を追加し、Codex Hooks の command hook から one-shot relay を実行できるようにした。
- `codex:clipboard` の `Get-Clipboard -Raw` 結果を PowerShell 側で UTF-8 bytes にし、Base64 として Node.js へ渡すようにした。
- Node.js 側で Base64 を UTF-8 復元して `answer.completed.body` に入れるようにした。
- relay smoke に日本語 clipboard 本文の回帰テストを追加した。
- 手動テスト手順に、長い日本語本文は `--text` 直渡しより clipboard / file watch を優先する注意を追加した。

## 手動テスト準備

- Host Bridge 起動: `cmd.exe /d /s /c npm run bridge:start -- --host=0.0.0.0 --port=8080`
- clipboard へ日本語本文を copy する。
- clipboard answer: `cmd.exe /d /s /c npm run codex:clipboard -- --summary "Codex clipboard answer"`
- file watch answer: `cmd.exe /d /s /c npm run codex:watch -- --file dist\codex-answer.txt --once`
- Codex session auto relay: `cmd.exe /d /s /c npm run codex:sessions -- --once --phase any`
- Codex session continuous relay: `cmd.exe /d /s /c npm run codex:sessions -- --phase final`
- Codex hook relay: `cmd.exe /d /s /c npm run codex:hook -- --bridge http://127.0.0.1:8080 --device-id m5stack-sample-001`
- hatch-pet asset: `cmd.exe /d /s /c npm run pet:asset -- --pet-dir %USERPROFILE%\.codex\pets\Mira --output firmware\include\pet_asset.local.h`
- event log: `Invoke-RestMethod -Uri http://127.0.0.1:8080/events`
- 期待結果: Core2 の Answer 画面に clipboard の日本語本文が文字化けせず表示され、header の pet avatar が hatch-pet asset として表示される。

## 未実施範囲

GRAY 実機、長時間運用、実 Codex App 内部 API 連携の手動テストはCodexでは未実施です。Core2 の clipboard 日本語表示と file watch の `Answer` 画面表示はユーザー目視で確認済みです。Codex App の非公開内部 API scraping は意図的に対象外とし、clipboard / stdin / file relay を実アダプタとして使います。
