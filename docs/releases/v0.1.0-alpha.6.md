# v0.1.0-alpha.6

## 位置づけ

`v0.1.0-alpha.6` は closed alpha prerelease です。`v0.1.0-alpha.5` の Dashboard / hatch-pet 連携を維持したまま、M5Stack 上の pet animation FPS を改善します。stable release ではありません。

## 追加実装

- firmware の pet animation を固定 320ms 更新から既定 `12fps` へ変更した。
- `display.settings_updated.display.animationFps` を追加し、Dashboard と `codex:display` から `4..20fps` の範囲で pet frame / bounce の更新速度を変更できるようにした。
- firmware loop の idle delay を `20ms` から `5ms` に短縮し、描画更新と入力処理の応答性を改善した。
- Dashboard の `Display` tab に `animation FPS` slider を追加し、M5Stack 表示プレビューにも FPS readout と preview animation duration を反映した。
- `tools/generate-pet-firmware-asset.py` が hatch-pet idle row の非空セルから frame 数を自動検出するようにし、Mira のような6フレーム pet を固定4フレームに切り詰めないようにした。
- sample、schema、mock device、manual test、QCDS、traceability、hardware evidence を `animationFps` に合わせて更新した。

## 検証結果

- `cmd.exe /d /s /c npm test`
- `cmd.exe /d /s /c npm run pet:asset -- --pet-dir %USERPROFILE%\.codex\pets\Mira --output firmware\include\pet_asset.local.h`
- `E:\DevEnv\PlatformIO\venv\Scripts\pio.exe run -d firmware -e m5stack-core2`
- `E:\DevEnv\PlatformIO\venv\Scripts\pio.exe run -d firmware -e m5stack-gray`
- `E:\DevEnv\PlatformIO\venv\Scripts\pio.exe run -d firmware -e m5stack-core2 -t upload --upload-port COM4`
- Playwright CLI: `http://127.0.0.1:18080/` で Dashboard が非blank表示され、`animation FPS` slider を `20fps` に変更したとき preview readout と CSS animation duration が更新されることを確認した。
- 起動中の 8080 Host Bridge は古い schema だったため、互換 fallback の `pet.updated.display.animationFps=12` を queue し、Core2 が poll して pending `0` になることを確認した。

## 手動テスト準備

- Host Bridge 起動: `cmd.exe /d /s /c npm run bridge:start -- --host=0.0.0.0 --port=8080`
- Dashboard display settings: `http://127.0.0.1:8080/` の `Display` tab で `pet display area=8/8`、text size、animation FPS を送信する。
- CLI display settings: `cmd.exe /d /s /c npm run codex:display -- --pet-scale 8 --ui-text-scale 2 --body-text-scale 2 --animation-fps 12`
- hatch-pet asset: `cmd.exe /d /s /c npm run pet:asset -- --pet-dir %USERPROFILE%\.codex\pets\Mira --output firmware\include\pet_asset.local.h`
- Core2 upload: `E:\DevEnv\PlatformIO\venv\Scripts\pio.exe run -d firmware -e m5stack-core2 -t upload --upload-port COM4`
- event log: `Invoke-RestMethod -Uri http://127.0.0.1:8080/events`
- 期待結果: Core2 の pet surface が固定ヘッダーテキストなしで表示され、`4fps`、`12fps`、`20fps` の順に pet frame / bounce の更新速度が変わる。

## 未実施範囲

GRAY 実機、長時間運用、実 Codex App 内部 API 連携の手動テストはCodexでは未実施です。Core2 への upload と Dashboard / CLI の FPS 設定送信経路は確認済みですが、`4fps`、`12fps`、`20fps` の物理画面での目視比較はユーザー手動確認対象です。
