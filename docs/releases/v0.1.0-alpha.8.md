# v0.1.0-alpha.8

## 位置づけ

`v0.1.0-alpha.8` は closed alpha prerelease です。現在の実装を MVP baseline として固定し、M5Stack の pet 表示、Dashboard GUI、Codex からの三択判断 request を正式リリースへ向けて整備します。stable release ではありません。

## 追加実装

- firmware の render FPS と pet frame / bounce 切替頻度を分離し、`motionStepMs` で高FPS時の小刻みな震えを抑えるようにした。
- M5Stack 実機の表示を Dashboard preview に合わせ、pet を背景として大きく表示し、Answer / Decision / Notification の本文を pet に重ねて描画するようにした。
- Dashboard preview が `/pet/current/manifest` と `/pet/current/spritesheet.webp` から現在の hatch-pet キャラを表示するようにした。
- Pet / Display 設定を `M5Stack 表示プレビュー` に統合し、pet display area、UI text size、body text size、render FPS、motion step を送信前に確認できるようにした。
- Dashboard の各 section に `Hide` / `View` を追加し、主要入力と slider に focus / hover tooltip を追加した。
- 環境構築と debug command は sidebar の `環境構築コマンド` modal へ移動した。
- Debug section に Decision 返信確認を統合した。
- `codex:decision`、`codex:decision:wait`、`/codex/decision` を追加し、Codex 側から M5Stack へ A/B/C の三択判断を求め、必要に応じて返信を command result として待てるようにした。
- `docs/formal-release-platform.md` を追加し、MVP baseline と正式リリース候補の platform workstream を整理した。

## 手動テスト準備

- Host Bridge 起動: `cmd.exe /d /s /c npm run bridge:start -- --host=0.0.0.0 --port=8080`
- Dashboard: `http://127.0.0.1:8080/`
- hatch-pet asset: `cmd.exe /d /s /c npm run pet:asset -- --pet-dir %USERPROFILE%\.codex\pets\Mira --output firmware\include\pet_asset.local.h`
- Core2 upload: `E:\DevEnv\PlatformIO\venv\Scripts\pio.exe run -d firmware -e m5stack-core2 -t upload --upload-port COM4`
- Display settings: `cmd.exe /d /s /c npm run codex:display -- --pet-scale 8 --ui-text-scale 2 --body-text-scale 2 --animation-fps 12 --motion-step-ms 280`
- Decision request: `cmd.exe /d /s /c npm run codex:decision -- --question "次の作業を選んでください" --a "進める" --b "修正する" --c "保留する"`
- Decision wait: `cmd.exe /d /s /c npm run codex:decision:wait -- --question "次の作業を選んでください" --a "進める" --b "修正する" --c "保留する" --wait-ms 300000`

期待結果:

- Core2 の pet surface が固定ヘッダーテキストなしで表示され、本文が pet に重なる。
- Dashboard preview と Core2 実機の layout が一致する。
- Dashboard preview は現在の hatch-pet キャラを表示する。
- `20fps / 280ms` でも pet が小刻みに震えず、本文や footer がちらつかない。
- M5Stack の A/B/C 返信が inbound log と `codex:decision:wait` の command result に戻る。

## 未実施範囲

GRAY 実機、長時間運用、実 Codex App 内部 API 連携の手動テストはCodexでは未実施です。Core2 への build / upload は実行対象ですが、Dashboard preview と実機 layout の一致、current pet preview、tooltip、section collapse、command modal、Decision wait、render FPS / motion step の物理画面目視はユーザー手動確認対象です。
