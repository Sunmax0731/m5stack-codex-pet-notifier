# v0.1.0-alpha.3

## 位置づけ

`v0.1.0-alpha.3` は closed alpha prerelease です。`v0.1.0-alpha.2` の LAN Host Bridge / firmware 経路に加え、Codex の返答本文を実際に Core2 へ送る Codex relay を追加します。stable release ではありません。

## 追加実装

- `src/codex-adapter/relay.mjs` を追加し、stdin / argument / clipboard / file watch から返答本文を送信可能にした。
- `src/codex-adapter/eventFactory.mjs` を追加し、`answer.completed`、`notification.created`、`prompt.choice_requested`、`pet.updated` の生成を共通化した。
- Host Bridge に `/codex/answer` と `/codex/notification` を追加した。
- `scripts/codex-relay-smoke.mjs` を追加し、relay CLI と `/codex/answer` の queue / poll を自動検証に含めた。
- Core2 firmware の touch 操作を拡張し、Choice row tap、footer touch、Answer footer paging、swipe paging を扱う。

## 手動テスト準備

- Host Bridge 起動: `cmd.exe /d /s /c npm run bridge:start -- --host=0.0.0.0 --port=8080`
- Codex answer: `cmd.exe /d /s /c npm run codex:answer -- --summary "Codex返答表示" --text "Core2に表示するCodex返答本文"`
- Clipboard answer: `cmd.exe /d /s /c npm run codex:clipboard -- --summary "Codex clipboard answer"`
- event log: `Invoke-RestMethod -Uri http://127.0.0.1:8080/events`

## 未実施範囲

GRAY 実機、長時間運用、実 Codex App 内部 API 連携の手動テストはCodexでは未実施です。Codex App の非公開内部 API scraping は意図的に対象外とし、clipboard / stdin / file relay を実アダプタとして使います。
